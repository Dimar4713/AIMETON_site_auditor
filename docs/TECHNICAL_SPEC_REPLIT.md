# Техническое задание для Replit: AIMETON Site Auditor

## 1. Назначение
Создать web-приложение AI-агентства, которое по URL главной страницы определяет профиль компании и предлагает 5–10 применимых AI-агентов с описанием функции и бизнес-пользы. После анализа пользователь продолжает диалог по результатам.

## 2. Пользовательский сценарий
1. Пользователь вводит публичный HTTP/HTTPS URL.
2. Нажимает «Проанализировать».
3. Система безопасно загружает главную страницу, извлекает видимый текст и заголовок.
4. RouterAI формирует структурированный анализ.
5. Интерфейс показывает профиль бизнеса, доказательства, рекомендации, риски и предположения.
6. Активируется чат, контекстом которого служит сохранённый результат анализа.

## 3. Интерфейс
- поле URL;
- кнопка «Проанализировать»;
- индикатор состояния и понятные ошибки;
- область результата;
- карточки 5–10 агентов: название, функция, польза, приоритет;
- чат после завершения анализа;
- адаптивность для desktop/mobile.

## 4. Backend API
- `GET /api/health`;
- `POST /api/analyze` с `{url}`;
- `POST /api/chat` с объектом анализа и историей сообщений.

## 5. Интеграция RouterAI
- OpenAI-compatible base URL: `https://routerai.ru/api/v1`;
- секрет `ROUTERAI_API_KEY` только в Replit Secrets;
- модель задаётся `ROUTERAI_MODEL`;
- ответ анализа валидируется Pydantic-схемой;
- при ошибке LLM включается ограниченный эвристический fallback.

## 6. Критерии приёмки
- приложение запускается в Replit одной командой;
- анализирует минимум три разные отрасли;
- выдаёт 5–10 рекомендаций;
- чат отвечает с учётом анализа;
- тесты `pytest -q` проходят;
- секрет отсутствует в Git-истории.
