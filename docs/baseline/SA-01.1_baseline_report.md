## Минимальный CI

Workflow `.github/workflows/baseline-ci.yml` выполняет:

1. установку Python 3.11;
2. установку `requirements.txt`;
3. фиксацию `pip freeze` в артефакт;
4. `pytest -q`;
5. импорт `app.main`;
6. проверку title/version;
7. локальный smoke `/api/health`;
8. проверку регистрации MCP route.

## Границы baseline

На данном шаге сознательно не выполняются:

- изменение поисковой логики;
- изменение MCP security/redirect;
- изменение Hunter;
- реальные вызовы RouterAI;
- реальные внешние OSINT-запросы;
- изменение data contracts.

Внешние endpoint-проверки `/api/analyze`, `/api/company-intelligence` и `/api/hunt` требуют контролируемой конфигурации providers и будут фиксироваться отдельно без публикации секретов.

## Известные ограничения исходного состояния

- до этого PR автоматический CI отсутствовал;
- транзитивные зависимости не были зафиксированы lock-файлом;
- внешние providers зависят от environment variables;
- локальная среда агента не имела DNS-доступа к GitHub, поэтому источником истины для прогона выбран GitHub Actions;
- прохождение или дефекты исходных тестов должны быть зафиксированы первым CI run, а не предположены заранее.
