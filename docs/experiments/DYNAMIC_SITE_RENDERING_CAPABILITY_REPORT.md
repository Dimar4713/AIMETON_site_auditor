# Отчёт об эксперименте Capability Scout: dynamic_site_rendering

**Проект:** AIMETON Site Auditor  
**Среда:** изолированная песочница ChatGPT  
**Решение:** способность не регистрировать в Capability Registry по результатам текущей среды.

## 1. Исходный Capability Gap

Статический загрузчик `httpx + BeautifulSoup` не извлекает содержимое сайтов, где основной текст создаётся JavaScript после загрузки HTML.

Требуемый контракт:

- вход: публичный HTTP/HTTPS URL;
- выход: итоговый URL, заголовок, видимый текст, размер HTML, идентификатор провайдера;
- тайм-аут;
- запрет private/loopback/link-local адресов;
- ограничение размера результата;
- воспроизводимое завершение браузерного процесса.

## 2. Реализованные кандидаты

### Кандидат A — Playwright

Созданы `playwright_fetcher.py`, контракт `RenderedPage`, fallback-resolver и тест динамической HTML-фикстуры.

Результат: **отклонён в текущей песочнице**.

Причина: установленный Python-пакет содержит бинарник Node, но не содержит `driver/package/cli.js`; встроенный бинарник также завершается ошибкой `Bad address`. Подмена на системный Node не помогает из-за отсутствующего JS-драйвера.

### Кандидат B — системный Chromium CLI

Создан `chromium_cli_fetcher.py`, использующий headless Chromium и `--dump-dom`.

Результат: **отклонён в текущей песочнице**.

Причина: Chromium запускается, но среда запрещает необходимые системные операции (`NETLINK socket: Permission denied`) и процесс не завершает рендеринг до тайм-аута.

## 3. Результаты тестов

- базовые тесты анализа — пройдены;
- определение необходимости JavaScript-рендеринга — пройдено;
- блокировка loopback URL — пройдена;
- контролируемая обработка отказа Playwright — пройдена;
- реальный браузерный contract test — `xfail`, потому что среда не предоставляет рабочий browser runtime.

Итог локального прогона: `6 passed, 1 xfailed`.

## 4. Архитектурный вывод

Capability-подход сработал корректно, потому что система:

1. выделила недостающую способность;
2. сформировала контракт;
3. рассмотрела два способа реализации;
4. создала адаптеры;
5. проверила их одинаковым тестом;
6. отличила дефект способности от ограничений среды;
7. отказалась регистрировать неподтверждённую способность.

Это предотвращает ложное включение инструмента в Runtime только на основании его наличия или популярности.

## 5. Решение

Статус: `rejected_in_current_sandbox`.

Способность следует повторно проверить в Replit или Codex-среде, где доступен поддерживаемый Playwright browser runtime.

Условия принятия:

- динамическая фикстура реально отрендерена;
- статические сайты не деградируют;
- процессы браузера завершаются;
- проверяются redirect targets;
- private/link-local subresources блокируются;
- лимиты CPU, памяти, времени и размера соблюдаются.

## 6. Следующий эксперимент

В Replit:

```bash
pip install -r requirements.txt
python -m playwright install chromium
pytest -q
```

После прохождения браузерного теста изменить Passport на `provisionally_accepted`, но только после отдельного SSRF/security теста.
