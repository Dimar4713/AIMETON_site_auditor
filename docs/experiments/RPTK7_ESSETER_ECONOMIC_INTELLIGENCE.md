# РПТК7 «Эссетер»: экономическая разведка AIMETON

## Статус

Экспериментальное архитектурное положение для ветки `experiment/dynamic-site-rendering`.

## Назначение

AIMETON Site Auditor рассматривается не как сервис подготовки формального аудита сайта, а как первый экспериментальный орган экономической разведки AIMETON, реализующий принцип РПТК7 «Эссетер».

Его задача — исследовать цифровую среду, обнаруживать доступные экономические возможности, квалифицировать их, переводить в управляемый коммерческий контур и передавать оператору или специализированному экземпляру AIMETON подготовленный объект действия.

## Терминологическое правило

В публичной технической архитектуре используются следующие нормативные обозначения:

- **обнаружение экономических сигналов** — технический эквивалент функции «нюхача»;
- **квалификация и формирование коммерческого контура** — технический эквивалент функции «загона»;
- **подготовка и передача коммерческой возможности** — технический эквивалент функции «подноса».

Формула «нюхач → загон → поднос» сохраняется как исходная функциональная метафора РПТК7, но не используется как основное название программных модулей.

## Функциональный цикл

```text
Картирование среды
→ обнаружение экономических сигналов
→ сбор подтверждающих признаков
→ квалификация возможности
→ подбор подходящего AI-решения
→ формирование коммерческого контура
→ подготовка демонстрационного сценария
→ передача коммерческой возможности
```

## 1. Картирование среды

Система формирует пространство исследования:

- компании Красноярска и Красноярского края;
- отрасли и сегменты;
- сайты, каталоги и публичные документы;
- продукты, услуги и способы продажи;
- доступные контакты и цифровые каналы.

Результат — структурированная карта потенциальных объектов экономической разведки.

## 2. Обнаружение экономических сигналов

Site Auditor ищет не просто технические дефекты сайта, а признаки неиспользованной ценности:

- сложный выбор товара или услуги;
- разрыв между ассортиментом и способом консультации;
- повторяющиеся вопросы клиентов;
- слабая квалификация заявок;
- потеря обращений вне рабочего времени;
- ручная подготовка расчётов и коммерческих предложений;
- высокий средний чек при слабом цифровом сопровождении;
- наличие задачи, которую может закрыть AI-инструмент.

```text
цифровой след компании
→ экономический сигнал
→ гипотеза потери денег или времени
→ возможная способность AIMETON
```

## 3. Сбор подтверждающих признаков

После обнаружения сигнала система:

- исследует сайт, каталог и публичные материалы;
- определяет бизнес-модель;
- оценивает зрелость цифрового контура;
- выявляет повторяющиеся клиентские сценарии;
- ищет доступное лицо, принимающее решение;
- оценивает возможный экономический эффект;
- фиксирует доказательства и уровень уверенности.

## 4. Квалификация возможности

Каждая найденная возможность получает оценку по следующим измерениям:

- выраженность проблемы;
- потенциальный экономический эффект;
- соответствие существующим способностям AIMETON;
- сложность создания решения;
- доступность лица, принимающего решение;
- ожидаемый чек;
- вероятность демонстрации и сделки;
- повторяемость решения для других компаний отрасли.

## 5. Подбор AI-решения

Система не предлагает всем компаниям один и тот же продукт. Она сопоставляет выявленную потребность с подходящей способностью:

| Экономический сигнал | Предлагаемое решение |
|---|---|
| Сложный каталог | AI-консультант по подбору |
| Непонятный выбор услуг | Навигатор услуг |
| Повторяющиеся вопросы | AI-первая линия |
| Необходим предварительный расчёт | AI-калькулятор и квалификатор |
| Долгая подготовка предложений | Генератор коммерческих предложений |
| Большой объём документации | AI-справочник |
| Неструктурированные обращения | AI-квалификатор лидов |

## 6. Формирование коммерческого контура

Открытая рыночная возможность последовательно сужается:

```text
рынок Красноярска
→ перспективная отрасль
→ подходящая компания
→ конкретная потеря или ограничение
→ подходящее AI-решение
→ демонстрационный сценарий
→ лицо, принимающее решение
→ рекомендуемое действие
```

На этом этапе сырая находка превращается в управляемую коммерческую ситуацию.

## 7. Подготовка и передача возможности

Результатом работы является не список сайтов и не отчёт о недостатках, а карточка коммерческой возможности:

```yaml
company:
  name: example
  website: https://example.ru
  industry: industrial_equipment

signals:
  - complex_product_selection
  - repetitive_customer_questions
  - weak_lead_qualification

opportunity:
  probable_loss: customer_dropout_before_contact
  confidence: 0.82
  expected_value_rub: 60000-120000

recommended_solution:
  type: ai_product_consultant
  demo_scenario: equipment_selection

contact:
  decision_maker: unknown
  recommended_channel: email_or_phone

next_action:
  prepare_personalized_demo: true
```

## 8. Экономическая роль в AIMETON

Site Auditor становится органом поиска внешней ценности, необходимой для:

- финансирования серверной инфраструктуры;
- оплаты моделей и API;
- развития новых способностей;
- создания специализированных экземпляров AIMETON;
- формирования повторяемых отраслевых продуктов;
- поддержания экономической устойчивости системы.

## 9. Архитектурные компоненты

Рекомендуемая техническая декомпозиция:

```text
Economic Intelligence Loop
├── Territory Mapper
├── Economic Signal Detector
├── Evidence Collector
├── Opportunity Qualifier
├── AI Solution Matcher
├── Commercial Context Builder
├── Demonstration Scenario Generator
└── Opportunity Delivery Interface
```

## 10. Каноническая формулировка

> AIMETON Site Auditor — первый экспериментальный орган РПТК7 «Эссетер», реализующий экономическую разведку посредством картирования цифровой среды, обнаружения экономических сигналов, квалификации возможностей, формирования управляемого коммерческого контура и передачи подготовленного объекта действия оператору или специализированному экземпляру AIMETON.

## 11. Практическая цель первого этапа

Первый контур применяется для исследования красноярских компаний и получения первых коммерческих заказов, которые должны финансировать развитие AIMETON и создание последующих специализированных экземпляров системы.
