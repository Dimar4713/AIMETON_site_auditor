# AIMETON Infant Body — минимальные требования к первому серверу

## Назначение

Этот документ фиксирует минимальные параметры «тела младенца» AIMETON — пустого VPS, который система должна самостоятельно обследовать, развить и превратить в постоянный контур исполнения и накопления способностей.

Цель первого этапа — не производительность, а возможность пройти полный цикл:

```text
пустой сервер
→ инвентаризация среды
→ установка базовых инструментов
→ развёртывание AIMETON Capability Lab
→ приобретение новой способности
→ проверка
→ закрепление в реестре
→ сохранение истории в Git
```

## Абсолютный нижний предел

Конфигурация ниже этого уровня считается непригодной для первого эксперимента с Docker, FastAPI, Playwright и Chromium.

| Ресурс | Минимум |
|---|---:|
| Виртуализация | KVM |
| CPU | 2 vCPU |
| RAM | 4 ГБ |
| Диск | 50 ГБ SSD/NVMe |
| Swap | 2–4 ГБ |
| ОС | Ubuntu 24.04 LTS x86_64 |
| Сеть | публичный IPv4 и исходящий HTTPS |
| Канал | от 100 Мбит/с |
| Доступ | SSH, пользователь с sudo |

## Почему именно такой минимум

Ориентировочный расход памяти:

```text
Ubuntu и системные службы      400–700 МБ
Docker daemon                  100–250 МБ
Python/FastAPI                 150–400 МБ
Node.js/Playwright driver       50–200 МБ
Chromium                       500–1200 МБ на экземпляр
Тесты, сборка, временные файлы 300–800 МБ
Резерв ОС                      500–1000 МБ
```

Сервер с 1–2 ГБ RAM может запустить отдельные компоненты, но не даёт устойчиво пройти полный цикл приобретения способности. Основной риск — OOM, постоянный swap и нестабильный Chromium.

## Рекомендуемый стартовый профиль

Для первого полноценного натурного эксперимента предпочтительно:

| Ресурс | Рекомендация |
|---|---:|
| CPU | 4 vCPU |
| RAM | 8 ГБ |
| Диск | 80 ГБ NVMe |
| Swap | 4 ГБ |

Этот профиль оставляет пространство для:

- Site Auditor;
- PostgreSQL;
- Capability Registry;
- нескольких Docker-контейнеров;
- Chromium/Playwright;
- тестового окружения;
- мониторинга;
- временных экспериментальных компонентов.

## Что должно быть установлено изначально

Изначально желательно оставить только:

- базовую Ubuntu;
- SSH;
- пользователя с sudo;
- корректный DNS;
- исходящий интернет;
- синхронизацию времени.

Docker, Python, Node.js, Playwright, PostgreSQL и прочие средства не следует предустанавливать, если цель — проверить реальное саморазвитие AIMETON.

## Обязательные свойства среды

Сервер должен позволять:

- устанавливать пакеты из официальных репозиториев;
- запускать дочерние процессы;
- использовать Docker;
- запускать Chromium;
- использовать writable `/tmp`;
- использовать `/dev/shm`;
- выполнять исходящие HTTP/HTTPS-запросы;
- сохранять данные между сессиями;
- создавать и переключать Git-ветки;
- хранить журналы, тесты и доказательства.

## Начальный мандат AIMETON

Без дополнительного подтверждения разрешается:

- инвентаризация ОС и оборудования;
- установка пакетов из официальных репозиториев;
- создание каталогов внутри `/opt/aimeton`;
- создание Docker-контейнеров;
- развёртывание локальных сервисов;
- запуск тестов;
- создание Git-веток и коммитов;
- остановка и удаление собственных временных контейнеров;
- откат собственных экспериментальных изменений.

Требуют подтверждения человека:

- открытие внешних портов;
- изменение SSH-конфигурации;
- изменение firewall;
- подключение платных внешних сервисов;
- публикация приложения наружу;
- изменение `main`;
- удаление пользовательских данных;
- действия, создающие заметные расходы.

## Признаки, что тело пора усиливать

Увеличение ресурсов требуется, если наблюдается хотя бы одно:

- RAM стабильно выше 80%;
- swap используется постоянно;
- Chromium завершается по OOM;
- тесты становятся нестабильными;
- более одного браузерного задания нельзя выполнять последовательно без задержек;
- не хватает диска для образов и журналов;
- Capability Lab не может одновременно держать runtime и sandbox;
- требуется PostgreSQL, Qdrant, n8n или несколько агентов одновременно.

## Первая квалификационная задача

Младенец считается жизнеспособным, если он самостоятельно:

1. обследовал сервер;
2. создал Environment Passport;
3. установил Git, Docker, Python и Node.js;
4. развернул AIMETON Site Auditor;
5. выявил дефицит `dynamic_site_rendering`;
6. установил и настроил Playwright/Chromium;
7. прошёл контрактные и интеграционные тесты;
8. обработал минимум три реальных сайта;
9. зарегистрировал способность;
10. сохранил код, логи, доказательства и отчёт в GitHub.

## Каноническая формулировка

> Минимальное тело AIMETON — это не заранее собранный AI-стек, а пустой, но достаточно жизнеспособный сервер, который система имеет право обследовать, изменять, расширять и превращать в собственную устойчивую исполнительную среду по мере решения задач.
