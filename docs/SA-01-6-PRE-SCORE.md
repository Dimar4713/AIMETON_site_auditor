# SA-01.6 — Обязательный pre-score режима «Охота»

## Порядок исполнения

```text
поисковый результат
→ проверка достаточности исходных признаков
→ объяснимый pre-score либо insufficient_data
→ minimum_pre_score
→ deep_audit_score
→ только затем загрузка сайта и глубокая обработка
```

Глубокая обработка запрещена до получения числового pre-score и достижения порога `deep_audit_score`.

## Формула

Базовое значение для результата с валидным доменом и текстом: `20`.

| Фактор | Значение | Unknown |
|---|---:|---|
| Соответствие региону | +25 | регион не задан или невозможно выделить токены |
| Коммерческий каталог / сложный выбор | +20 | текст результата отсутствует |
| Содержательная коммерческая сложность | +15 | текст результата отсутствует |
| Соответствие заданному focus | +10 | focus не задан |
| Домен `.ru` | +5 | домен отсутствует |

Итог ограничивается диапазоном `0..100`.

Количество ссылок, число повторов результата и позиция в поисковой выдаче в формулу не входят.

## Явные состояния

- `calculated` — pre-score рассчитан и сопровождается таблицей факторов;
- `insufficient_data` — расчёт невозможен, `preliminary_score = null`, отсутствующие факторы сохраняются как `null`, а не как нули.

## Пороги

- ниже `minimum_pre_score` — кандидат отбрасывается;
- от `minimum_pre_score` до `deep_audit_score` — кандидат остаётся в наблюдении без загрузки сайта;
- не ниже `deep_audit_score` — разрешается загрузка сайта и глубокая обработка;
- `insufficient_data` — глубокая обработка запрещена независимо от числовых порогов.

## Объяснимость

Каждый возвращённый кандидат содержит:

- `pre_score_status`;
- `preliminary_score`;
- `pre_score_factors`;
- `reasons`;
- `deep_analysis_performed`.

Таким образом, порядок ранжирования и факт запуска глубокой обработки восстанавливаются из ответа API.
