# AIMETON Site Auditor

Экспериментальный орган экономической разведки AIMETON. Работает в двух режимах:

1. анализ конкретного сайта по URL;
2. автономная «охота» по заданной территории и параметрам поиска.

## Принцип работы охоты

Пользователь задаёт только рамку:

- регион;
- дополнительную зону поиска;
- приоритетные отрасли и признаки, если они нужны;
- лимиты глубины и числа целей.

Далее агент самостоятельно:

1. формирует поисковые запросы;
2. обнаруживает сайты компаний;
3. исключает агрегаторы, социальные сети и дубли;
4. проверяет соответствие территории;
5. выполняет предварительную фильтрацию;
6. классифицирует бизнес;
7. оценивает коммерческую возможность;
8. ранжирует цели;
9. глубоко прорабатывает лучшие цели;
10. формирует пакет коммерческого действия.

Каноническая техническая цепочка:

```text
обнаружение экономических сигналов
→ квалификация и формирование коммерческого контура
→ подготовка и передача коммерческой возможности
```

## API

### Анализ конкретного сайта

```http
POST /api/analyze
Content-Type: application/json

{"url":"https://example.ru"}
```

### Запрос на охоту

```http
POST /api/hunt
Content-Type: application/json

{
  "region": "Красноярск и Красноярский край",
  "search_zone": "Красноярская агломерация",
  "industries": ["отопление и вентиляция", "промышленное оборудование"],
  "focus": ["сложный подбор", "повторяющиеся консультации"],
  "max_candidates": 100,
  "deep_audit_score": 60,
  "output_limit": 25
}
```

Для автономного веб-поиска задайте переменную окружения:

```text
SEARXNG_BASE_URL=https://your-searxng-instance.example
```

Поисковый провайдер вынесен из ядра: текущая реализация использует JSON API SearXNG, но модуль можно заменить другим разрешённым поисковым API.

## Запуск

1. Импортируйте репозиторий в Replit или разверните на Linux-сервере.
2. Добавьте `ROUTERAI_API_KEY`.
3. При необходимости задайте `ROUTERAI_MODEL`.
4. Для режима охоты задайте `SEARXNG_BASE_URL`.
5. Выполните `pip install -r requirements.txt`.
6. Запустите:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 5000
```

## Безопасность

- ключи хранятся только в переменных окружения;
- SSRF-защита блокирует локальные и служебные IP;
- ограничены тип, размер и время загрузки страницы;
- LLM не получает право самостоятельно выполнять внешние коммерческие действия;
- поиск и анализ не означают автоматическую массовую рассылку;
- перед внешним контактом требуется отдельный разрешённый контур действия.

## Тесты

```bash
pytest -q
```

Изменения экспериментальной ветки требуют повторного прогона тестов и проверки на реальных сайтах перед публикацией.
